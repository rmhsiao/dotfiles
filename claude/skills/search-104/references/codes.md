# 104 篩選代碼（非 JSON 部分）

`area` 和 `jobcat` 從 `references/Area.json` / `references/JobCat.json` 解析（見 SKILL.md 的 Step 2，由 subagent 處理）。

以下是其他小型篩選旗標，值少而固定，直接使用。

## isnew (更新時間，天數)

| 範圍 | code |
|------|------|
| 本日最新 | `0` |
| 三日內 | `3` |
| 一週內 | `7` |
| 兩週內 | `14` |
| 一個月內 | `30` |

Default for "recent" / "最近": `7`。

## jobexp (經歷要求 — 年數)

| 範圍 | code |
|------|------|
| 1 年以下 | `1` |
| 1–3 年 | `3` |
| 3–5 年 | `5` |
| 5–10 年 | `10` |
| 10 年以上 | `99` |

多選以 `,` 連接。例如「3 年以上資深」: `5,10,99`。

## edu (學歷要求)

| 學歷 | code |
|------|------|
| 高中職以下 | `1` |
| 高中職 | `2` |
| 專科 | `3` |
| 大學 | `4` |
| 碩士 | `5` |
| 博士 | `6` |

## remoteWork (上班型態)

| 型態 | code |
|------|------|
| 完全遠端 | `1` |
| 部分遠端 | `2` |

不帶此參數 = 全部。

## order / asc (排序)

| order | 意義 |
|-------|------|
| `1` | 符合度 (default) |
| `2` | 日期 |
| `13` | 待遇 |

| asc | 方向 |
|-----|------|
| `0` | 由大到小 (新/高薪先) |
| `1` | 由小到大 |

## zone (公司類型)

| 類型 | code |
|------|------|
| 上市上櫃 | `16` |
| 外商一般 | `5` |
| 外商資訊 | `4` |

## 備註

這些值都是 104 長年穩定的，即使未來 Area / JobCat 的分類有更動，這些旗標的對應值幾乎不會變。如果真的變了，用一個樣本職缺的搜尋 URL 反查即可。
